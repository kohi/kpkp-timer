## 概要

KPKP-Timer は、ワークショップ・プレゼンテーション用に設計された多機能タイマーWebアプリです。
Progressive Web App (PWA) として実装されており、ブラウザから直接利用できるほか、端末へのインストールも可能です。

---

## アーキテクチャ

| 項目 | 内容 |
|------|------|
| 種別 | PWA (Progressive Web App) |
| 構成 | 静的HTML/CSS/JavaScript（サーバー不要） |
| 主要ファイル | `index.html`, `js/script.js`, `css/style.css`, `sw.js` |
| ストレージ | `localStorage`（設定・プリセット永続化） |
| オフライン対応 | Service Worker によるキャッシュ戦略 |
| Service Worker バージョン | `kpkp-timer-v2.0.1` |

---

## 画面構成

### メイン画面

タイマー動作中のメイン表示画面。2つのモードを切り替えて使用する。

#### タイマーモード
- **人数カウンター**: 現在の発表者番号を表示（例: `1 人目`）
- **メインタイマー表示**: 現在のタイマー残り時間または経過時間を大きく表示（`MM:SS` 形式）
- **ワーク名表示**: 現在実行中のタイマー名（例: T1, T2）
- **残り時間表示**: 全タイマーの合計残り時間
- **操作ボタン**: スタート / 一時停止 / リセット / スキップ

#### ストップウォッチモード
- **表示**: 経過時間（`HH:MM:SS` 形式）
- **操作ボタン**: 開始/一時停止 / リセット

#### 共通UI
- **モード切り替えボタン**: タイマーモード ↔ ストップウォッチモード
- **設定ボタン（FAB）**: 設定画面へ遷移（⚙️ ボタン）

---

### 設定画面

全設定を行う画面。「保存してメイン画面に戻る」で設定を保存・反映。

## 機能仕様

### 1. タイマー機能

#### 基本仕様
- タイマーは最大 **30個** まで登録可能
- 各タイマーには **名前**、**時間（秒）**、**インターバル時間（秒）** を設定
- **タイマータイプ**（全タイマー共通）:
  - `カウントダウン`: 設定時間から0秒に向かってカウント
  - `カウントアップ`: 0秒から設定時間に向かってカウント

#### タイマー実行フロー
1. 登録されたタイマーを順番に実行
2. 各タイマー終了後、インターバル時間（設定時）があればインターバルへ移行
3. インターバル終了後、次のタイマーへ自動移行
4. 全タイマー完了後、アラーム音を鳴らして停止

#### タイマー操作
| 操作 | 説明 |
|------|------|
| スタート | タイマー開始（最初のタイマーから）|
| 一時停止 | タイマーを一時停止 |
| リセット | 全タイマーを初期状態に戻す |
| スキップ | 現在のタイマーをスキップして次へ |

#### タイマー精度
- 100ms間隔でインターバル更新（`TIMER_TICK_INTERVAL_MS = 100`）
- 開始時刻と経過時間の差分で実際の経過時間を計算

---

### 2. 人数カウンター機能

ワークショップでの発表者番号管理に使用する機能。

| 設定 | 説明 |
|------|------|
| カウンター表示 ON/OFF | メイン画面での表示・非表示を切り替え |
| 自動カウントアップ | タイマー1周ごとに自動で番号を+1 |
| カウンターラベル | 番号の後ろに表示するテキスト（デフォルト: `人目`） |
| 手動カウントアップ | `Ctrl + C` キーで手動で+1 |

---

### 3. プリセット機能

タイマー設定をプリセットとして保存・呼び出しできる機能。

- **保存先**: `localStorage`（ブラウザに永続保存）
- **操作**:
  - プリセット保存: 任意の名前を入力して保存
  - プリセット読み込み: ドロップダウンから選択して即時反映
  - プリセット削除: 選択中のプリセットを削除
- **注意**: ブラウザのキャッシュクリア・シークレットモードでは削除される
- **バックアップ**: CSVエクスポートを推奨

---

### 4. CSV機能

| 機能 | 説明 |
|------|------|
| CSVインポート | CSVファイルからタイマー設定を一括インポート |
| CSVエクスポート | 現在のタイマー設定をCSVファイルとして書き出し |
| CSVテンプレートDL | インポート用のCSVテンプレートをダウンロード |

### 5. 音声機能

#### 音声種別

| 種別 | 説明 |
|------|------|
| BGM | タイマー動作中に流れるバックグラウンドミュージック（ループ再生） |
| アラーム音 | タイマー終了時に鳴る音 |
| インターバル終了音 | インターバル終了時に鳴る音 |
| ショートビープ音 | 残り時間警告時に鳴る短い音 |

#### 音声設定

| 設定 | 説明 |
|------|------|
| マスター音声 ON/OFF | 全音声を一括でON/OFF |
| BGM ON/OFF | BGMの再生・停止 |
| タイマー終了音 ON/OFF | アラーム音の有効・無効 |
| インターバル終了音 ON/OFF | インターバル音の有効・無効 |
| 残り時間ビープ音 ON/OFF | 警告ビープ音の有効・無効 |
| ビープ音秒数 | ビープ音を鳴らす残り秒数（カンマ区切り、デフォルト: `10,5,3`） |
| 内蔵音源を使用 | ファイル未設定時にWeb Audio APIの内蔵音源を使用 |

#### 音声ファイル
- ユーザーがローカルファイルを選択してアップロード可能
- 対応形式: audio/* (mp3, wav, ogg, m4a 等)
- ファイルはBase64エンコードして`localStorage`に保存

#### Web Audio API
- カスタム音源未設定時に内蔵のビープ音を生成
- サイン波オシレーター使用（デフォルト周波数: 440Hz）

---

### 6. 視覚警告機能

残り時間に応じて背景色を自動変化させる機能。

| 設定 | 説明 |
|------|------|
| 残り時間背景色変化 ON/OFF | 視覚警告の有効・無効 |
| 背景色変化秒数 | 色変化のトリガー秒数（カンマ区切り、デフォルト: `10,5,3`） |

- カウントアップモードでは「目標時間との差分」で判定
- 複数の警告レベルを設定可能（秒数が小さいほど強い警告色）

---

### 7. 表示設定

| 設定 | 説明 |
|------|------|
| メインタイマー表示 ON/OFF | 大きなタイマー数字の表示・非表示 |
| 残り時間表示 ON/OFF | サブエリアの残り時間表示・非表示 |
| ワーク名大表示 ON/OFF | タイマー名の表示・非表示（OFFの場合は人数カウント部分に統合） |

---

### 8. 壁紙・カラー設定

#### 背景画像
- ローカルの画像ファイルを背景として設定可能
- 対応形式: image/* (png, jpg, gif, webp 等)
- Base64エンコードして`localStorage`に保存

#### フォント色設定

| 設定 | デフォルト値 |
|------|-------------|
| メインタイマー色 | `#2c3e50` |
| サブ表示色 | `#2c3e50` |
| カウンター色 | `#2c3e50` |

各色にリセットボタンあり。

---

### 9. レイアウト設定

| モード | 説明 |
|--------|------|
| コンパクト（PC向け） | デスクトップ向けレイアウト（デフォルト） |
| モバイル最適化 | スマートフォン向けレイアウト |

- レスポンシブブレークポイント: `768px`（`MOBILE_BREAKPOINT`）

---

### 10. 通知機能

- Web Push通知によるタイマー完了通知
- ユーザーによる許可が必要
- タイマー完了時にブラウザ通知を送信
- 通知にはアプリを開くアクションボタンを含む

## キーボードショートカット

| キー | 動作 |
|------|------|
| `Space` | タイマーモード: スタート/一時停止トグル / ストップウォッチモード: 開始/停止 |
| `R` | リセット |
| `S` / `N` | スキップ（タイマーモードのみ） |
| `T` | メイン画面/設定画面の切り替え |
| `O` | 設定画面を開く |
| `M` | マスター音声ON/OFFトグル |
| `L` | レイアウトモード切り替え（コンパクト/モバイル） |
| `D` | タイマーモード/ストップウォッチモードの切り替え |
| `Ctrl + C` | カウンターを手動で+1（タイマーモードのみ） |

## URL制御

### クエリパラメータ
- `?screen=settings` : 起動時に設定画面を直接開く

### URLハッシュ
- `?screen=settings#timer-settings` : 設定画面の特定セクションにスクロール

### PostMessage API（外部制御）
iframeで埋め込んだ際に親ページからJavaScriptで操作可能。

| action | 動作 |
|--------|------|
| `start` | タイマー/ストップウォッチを開始 |
| `pause` | 一時停止 |
| `reset` | リセット |
| `skip` | スキップ（タイマーモードのみ） |
| `settings` | 設定画面を開く |
| `toggleMode` | タイマー/ストップウォッチモード切り替え |

## データ永続化

すべての設定は`localStorage`に保存される。

| キー | 内容 |
|------|------|
| `timerAppPresets` | ユーザー保存プリセット一覧 |
| （設定データ） | タイマー設定、音声ファイル（Base64）、背景画像（Base64）等 |

**注意**: 音声・画像ファイルはBase64変換してlocalStorageに保存するため、大きなファイルはブラウザの容量制限に注意。

## 制約・注意事項

- `localStorage` の容量制限により、大きな音声・画像ファイルの保存には注意が必要
- シークレットモードやブラウザのキャッシュクリアでプリセット・設定が消える
- Web Audio APIはブラウザのユーザーインタラクション後に初期化される
- タイマー最大登録数: 30個（`TIMER_MAX_COUNT = 30`）
- 音声再生はブラウザの自動再生ポリシーに依存するため、最初の操作後に有効になる
